# Проект OOP Design C++

Сейчас класс в Zoo не имеет никаких методов, и из за этого тесты не проходят, нам нужно чтобы они проходили и для этого нужно дописать класс Zoo

## Демонстрируемые принципы OOP

### 1. **Инкапсуляция**
- Приватные атрибуты членов
- Публичные методы доступа
- Контролируемый доступ через getters

### 2. **Наследование**
- `Animal` - абстрактный базовый класс
- Конкретные реализации: `Dog`, `Cat`, `Bird`
- `Zoo` класс с композицией

### 3. **Полиморфизм**
- Чистые виртуальные методы `makeSound()` и `eat()`
- Разные реализации в каждом классе животного
- Переопределение методов с `override`

### 4. **Абстракция**
- Использование чистых виртуальных функций
- Скрытие деталей реализации
- Определение интерфейсов через базовые классы

### 5. **Композиция**
- `Zoo` класс управляет `std::vector` объектов `Animal`
- Использование `std::shared_ptr` для безопасного управления памятью
- Отношение "имеет" (Zoo имеет Animals)

### 6. **Принцип единственной ответственности**
- Каждый класс имеет одну четкую цель
- `Animal`: Базовое поведение животного
- `Dog/Cat/Bird`: Поведение, специфичное для вида
- `Zoo`: Управление коллекцией

## Возможности

### Классы животных
- **Dog**: Обучается трюкам и может их выполнять
- **Cat**: Управляет настроением, может царапать предметы
- **Bird**: Может летать на разные высоты (кроме пингвина!)

### Управление зоопарком
- Добавление/удаление животных
- Кормление всех животных одновременно
- Физическая нагрузка для всех животных
- Поиск животных по имени
- Получение количества животных по виду
- Отображение информации обо всех животных

## Требования

- C++17 или более поздняя версия
- CMake 3.10 или более поздняя версия
- Компилятор, поддерживающий C++17 (GCC, Clang, MSVC)

## Характеристики качества кода

### Комментарии в документе
Все классы и методы включают подробные документирующие комментарии в стиле Doxygen.

```cpp
/**
 * Накормить животное.
 */
virtual void eat(const std::string& food) = 0;
```

### Правильное управление памятью
Использование умных указателей (`std::shared_ptr`, `std::unique_ptr`) для автоматического управления памятью.

```cpp
auto dog = std::make_shared<Dog>("Rex", 5, "Golden Retriever");
auto zoo = std::make_unique<Zoo>("My Zoo", "New York");
```

### STL контейнеры
Использование стандартных контейнеров для управления коллекциями.

```cpp
std::vector<std::shared_ptr<Animal>> animals;
```

### Виртуальные функции
Правильное использование виртуальных функций и полиморфизма.

```cpp
virtual std::string makeSound() const override;
virtual ~Animal() = default;
```

### Модульные тесты
Полное покрытие тестами использованием Google Test Framework:
- Инициализация объектов
- Поведение методов
- Граничные случаи
- Управление коллекциями

## Пример использования

```cpp
#include "Dog.h"
#include "Cat.h"
#include "Zoo.h"
#include <memory>

int main() {
    // Создать зоопарк
    auto zoo = std::make_unique<Zoo>("Мой парк", "Нью-Йорк");
    
    // Создать животных
    auto dog = std::make_shared<Dog>("Rex", 5, "Golden Retriever");
    auto cat = std::make_shared<Cat>("Whiskers", 3, "Orange Tabby");
    
    // Добавить в зоопарк
    zoo->addAnimal(dog);
    zoo->addAnimal(cat);
    
    // Использовать животных
    dog->teachTrick("Sit");
    std::cout << dog->performTrick("Sit") << std::endl;
    
    // Управлять зоопарком
    zoo->feedAll();
    zoo->displayAnimals();
    
    return 0;
}
```
